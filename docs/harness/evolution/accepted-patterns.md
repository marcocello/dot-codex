# Accepted Harness Patterns

Optional maintenance file. Use only after a harness pattern has proved useful through repeated use, tests, rollout evidence, or real proof results.

Each entry should name the evidence, the owning component, and the reason it should remain part of the harness.

## Template

```md
## <pattern>

- Component: <AGENTS.md|skill|script|docs|test|config|memory>
- Evidence: <proof, rollout, test, or repeated successful use>
- Why keep: <short rationale>
- Last checked: YYYY-MM-DD
```
